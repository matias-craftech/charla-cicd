Lo recorremos estación por estación.

CI · integración continua

Cada cambio se integra seguido

Cada push o PR dispara build + tests automáticos.

  • Resuelve el integration hell y el “en mi máquina anda”
  • Lo contrario de la lista de archivos en el chat: integra una máquina, todo el tiempo, y avisa al toque si algo rompe
CD · entrega continua

Siempre listo para release

Todo lo que pasa CI queda a un botón de producción. Desplegar = una decisión humana.

  • La release deja de ser un evento de terror
  • Adiós al .rar armado a mano un viernes.
CD · despliegue continuo · el otro CD

Se saca el botón humano

Lo que pasa el pipeline va a producción solo.

  • Requiere tests sólidos · monitoreo · rollback
  • Exige confianza, no valentía: la máquina decide porque antes le construimos las redes de seguridad
la bisagra · la que todos confunden

Los dos CD, lado a lado

continuous

Delivery

· Listo para desplegar
· Un humano decide
continuous

Deployment

· Desplegado solo
· La máquina decide

La diferencia es una decisión de negocio, no técnica.

matices

Es una escalera, no un interruptor

Integrás → entregás → a veces desplegás solo.

  • El deployment continuo no siempre se quiere
  • A veces quedarse en Delivery es lo correcto: regulación, sistemas críticos, aprobaciones
  • Honestidad > dogma
beneficios

Feedback rápido · menos miedo
releases aburridos · foco en features
